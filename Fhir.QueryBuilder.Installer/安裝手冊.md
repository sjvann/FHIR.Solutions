# FHIR Query Builder — 建置、發布與安裝手冊

本文件供 **交付／建置人員** 與 **終端使用者** 使用：前半為官方發布流程與指令，後半為客戶端安裝與操作說明。

---

## 第一部分：建置與發布（交付工程師）

### 環境需求

| 項目 | 說明 |
|------|------|
| 作業系統（建置機） | Windows x64（建置安裝程式與 `win-x64` 自包含主機） |
| .NET SDK | **10.0**（與方案 `net10.0` 一致） |
| Python | **3.x**（執行建置腳本；僅使用標準函式庫） |
| Inno Setup | **6.x**（需 `ISCC.exe`，用於編譯 `.exe` 安裝程式） |

Inno Setup 下載：<https://jrsoftware.org/isdl.php>

---

### 最終發布指令（Windows 安裝程式，建議）

於儲存庫根目錄或下列目錄開啟終端機，建置完成後安裝檔位於 **`artifacts\FHIRQueryBuilder-Setup-<版本>.exe`**。

```powershell
cd Fhir.QueryBuilder.Installer
py build_installer.py
```

等同於（若已將 `python` 加入 PATH）：

```powershell
cd Fhir.QueryBuilder.Installer
python build_installer.py
```

指定組態（通常不需修改）：

```powershell
py build_installer.py --configuration Release
```

**產出說明**

| 產出 | 路徑 |
|------|------|
| 客戶安裝程式 | `artifacts\FHIRQueryBuilder-Setup-1.0.0.exe`（版本號見 `FHIRQueryBuilder.iss` 內 `#define MyAppVersion`） |
| 未打包之安裝內容（除錯／手動打包） | `artifacts\FHIRQueryBuilderInstallerStaging\app\` |

若未安裝 Inno Setup，腳本仍會完成 **staging**，並提示以 Inno IDE 或 `ISCC.exe` 手動編譯 `FHIRQueryBuilder.iss`。

---

### 其他發布方式（無安裝程式）

僅產出 **Blazor WebAssembly** 靜態網站與 ZIP（適 IIS、nginx、物件儲存靜態網站等；**不要求**客戶安裝本機啟動器）：

```powershell
cd Fhir.QueryBuilder.App.Blazor
py publish_product.py
```

自訂輸出根目錄：

```powershell
py publish_product.py -o D:\publish-out
```

**產出結構**（預設根目錄為 `artifacts\Fhir.QueryBuilder.App.Blazor\<時間戳>\`）：

| 項目 | 說明 |
|------|------|
| `full\` | 完整 `dotnet publish`（含 `web.config`、`wwwroot`），適 IIS／Azure App Service |
| `static\` | 僅 **`wwwroot` 內容**，適 CDN／純靜態託管 |
| `*-IIS.zip` / `*-Static.zip` | 上述兩種封裝之 ZIP |

---

### 對照：僅使用 .NET CLI（進階）

不安裝 Python 時，可手動執行下列指令達成與 `build_installer.py` 類似之 staging（仍須自行執行 Inno Setup 包裝）：

```powershell
dotnet publish Fhir.QueryBuilder.App.Blazor\Fhir.QueryBuilder.App.Blazor.csproj -c Release -o artifacts\FHIRQueryBuilderInstallerStaging\blazor-publish

dotnet publish Fhir.QueryBuilder.App.LocalHost\Fhir.QueryBuilder.App.LocalHost.csproj -c Release -r win-x64 --self-contained true -p:PublishReadyToRun=true -o artifacts\FHIRQueryBuilderInstallerStaging\app

xcopy /E /I /Y artifacts\FHIRQueryBuilderInstallerStaging\blazor-publish\wwwroot artifacts\FHIRQueryBuilderInstallerStaging\app\wwwroot
```

接著於 `Fhir.QueryBuilder.Installer` 目錄執行 Inno Setup 編譯 `FHIRQueryBuilder.iss`。

## 第二部分：終端使用者安裝與操作

### 系統需求

| 項目 | 說明 |
|------|------|
| 作業系統 | Windows **10／11**，**64 位元**（x64） |
| 瀏覽器 | **Chrome、Edge、Firefox** 等現代瀏覽器（需支援 WebAssembly） |
| 網路 | 查詢 FHIR 伺服器及 OAuth 時需可連線網際網路（依貴單位資安政策） |
| 管理員權限 | **安裝程式預設需提升權限**（安裝至 `Program Files`） |

本產品利用 **本機 HTTP 服務**（`127.0.0.1`）提供網頁介面；**不需**另行安裝 .NET Runtime。

---

### 安裝步驟

1. 取得交付之 **`FHIRQueryBuilder-Setup-<版本>.exe`**。
2. 連按兩下執行；若出現使用者帳戶控制（UAC），請選擇「是」。
3. 依精靈選擇安裝目錄（預設：`C:\Program Files\FHIR Query Builder`）。
4. 可選：建立桌面捷徑。
5. 完成後可勾選「啟動程式」結束精靈。

安裝目錄內含 **`安裝手冊.md`**（本檔）與程式檔案。

---

### 啟動與瀏覽

1. 自「開始」選單或桌面捷徑執行 **FHIR Query Builder**（實際處理程式為 **`FHIRQueryBuilder.exe`**）。
2. 程式會於 **`http://127.0.0.1:8797`** 啟動本機網頁服務（預設埠），並嘗試開啟預設瀏覽器。
3. 請勿關閉出現的黑色主控台視窗；關閉即停止服務。若要停止使用，於該視窗按 **Ctrl+C** 或直接關閉視窗。

若埠號已被其他程式占用，請見下一節修改設定。

---

### 設定檔（選用）

安裝目錄下的 **`appsettings.json`** 可調整本機服務：

```json
{
  "LocalHost": {
    "Urls": "http://127.0.0.1:8797",
    "LaunchBrowser": true
  }
}
```

| 鍵值 | 說明 |
|------|------|
| `LocalHost:Urls` | 監聽位址與埠號；僅需單一網址時維持一字串即可。 |
| `LocalHost:LaunchBrowser` | 是否於啟動時自動開啟瀏覽器（`true`／`false`）。 |

FHIR 伺服器預設位址、OAuth（SMART）等設定見 **`wwwroot\appsettings.json`**（應用程式讀取之組態）。修改後請重新啟動 **`FHIRQueryBuilder.exe`**。

---

### 解除安裝

透過 **Windows 設定 → 應用程式 → 已安裝的應用程式**，選擇 **FHIR Query Builder** 並解除安裝；或使用「新增或移除程式」。

---

### 常見問題

**Q：瀏覽器顯示無法連線？**  
確認 **`FHIRQueryBuilder.exe`** 是否仍在執行、主控台視窗是否關閉；防火牆一般不會阻擋僅監聽 `127.0.0.1` 的本機連線。

**Q：埠號衝突？**  
變更 `appsettings.json` 內 `LocalHost:Urls` 之埠號後重新啟動程式。

**Q：是否須安裝 IIS？**  
**不需要。** 安裝套件已內含本機網頁伺服器。

**Q：資料是否經過網際網路？**  
介面於本機瀏覽器執行；查詢 FHIR 伺服器時，請求會依您設定的伺服器 URL 送出（可能為公網或院內網路）。

**Q：畫面一直轉圈，開發者工具（F12）出現 `/_framework/` 底下 `.dat` 檔 404？**  
較舊版本之本機啟動器未將 `.dat` 等 WASM 附屬檔視為可提供的靜態檔。請向交付窗口索取 **已修正之安裝程式** 並重新安裝；或請交付端重新執行 `build_installer.py` 建置後再安裝。
